Pendahuluan

KlatOS adalah sistem operasi modern berbasis JowoKernel, sebuah kernel monolitik 64-bit yang dikembangkan sepenuhnya dari nol (from scratch). Seluruh komponen inti sistem, mulai dari kernel, manajemen memori, scheduler, virtual file system, runtime userspace, sistem grafis, hingga desktop environment dirancang dan diimplementasikan secara mandiri tanpa menggunakan kode kernel dari sistem operasi lain.

Tujuan utama KlatOS adalah menghadirkan platform desktop modern yang ringan, cepat, stabil, dan kompatibel dengan standar POSIX sehingga mampu menjalankan aplikasi modern melalui antarmuka yang familiar bagi pengembang.

Key Facts

  • Kernel monolitik 64-bit
  • Dikembangkan from scratch
  • Kompatibel POSIX
  • Event-driven architecture
  • Graphics stack berbasis DRM
  • Desktop environment modular

Filosofi

Modern Architecture

Seluruh subsistem menggunakan arsitektur modern berbasis object-oriented dan event-driven sehingga setiap komponen memiliki tanggung jawab yang jelas, mudah dipelihara, dan dapat dikembangkan secara independen.

Performance First

Seluruh jalur kritis dirancang untuk meminimalkan overhead:

  • Zero-copy Virtual Memory Object (VMO)
  • Event-driven WaitQueue
  • Non-blocking I/O
  • Shared Memory IPC
  • DRM Dumb Buffer
  • Page Flipping

POSIX Compatibility

KlatOS mengimplementasikan antarmuka POSIX sehingga berbagai aplikasi dan pustaka dapat dipindahkan dengan perubahan seminimal mungkin.

Clean Architecture

Kernel tidak dibangun dari kumpulan patch, tetapi menggunakan desain modular yang memiliki batas antarsubsistem yang jelas.

Manajemen Proses

JowoKernel mengelola proses dan thread dengan fitur lengkap:

  • Multi-process Scheduler
  • Multi-thread Scheduler
  • Context Switching
  • Process Groups
  • Sessions & Job Control
  • Signal Delivery
  • Process Waiting
  • Orphan Process Handling
  • Zombie Process Management

Manajemen Memori

Sistem manajemen memori JowoKernel:

  • Physical Memory Manager (PMM) — mengelola alokasi frame fisik
  • Virtual Memory Manager (VMM) — mengelola page table dan mapping
  • Kernel Heap — alokasi memori dinamis untuk kernel
  • Slab Allocator — alokasi efisien untuk objek berukuran kecil
  • Virtual Memory Object (VMO) — abstraksi zero-copy untuk mapping
  • Memory Mapping — mmap/munmap untuk userspace
  • Shared Memory — IPC berbasis shared memory
  • Guard Pages — deteksi stack overflow
  • Copy-on-Write ready infrastructure

Filesystem

Virtual File System (VFS) dan filesystem yang tersedia:

  • Virtual File System (VFS) — layer abstraksi untuk semua filesystem
  • TmpFS — filesystem berbasis RAM untuk file sementara
  • ProcFS — filesystem virtual untuk informasi proses
  • DeviceFS — filesystem untuk device nodes
  • File Descriptor Table
  • Symbolic Links & Hard Links
  • Permission Management
  • Path Resolution

IPC (Inter-Process Communication)

Implementasi IPC menggunakan Unix Domain Socket modern:

  • Pipes
  • Unix Domain Socket (AF_UNIX)
  • Socket Pair
  • Stream Socket
  • Non-blocking Socket
  • Shared Memory Transfer
  • Zero-copy Buffer Sharing
  • File Descriptor Passing (SCM_RIGHTS)
  • Epoll Integration

Event Infrastructure

Salah satu perubahan arsitektur terbesar pada JowoKernel adalah transisi menuju model event-driven. Setiap objek filesystem memiliki WaitQueue internal.

Objek seperti Pipe, Unix Socket, Terminal, DRM Device, dan Input Device dapat membangunkan thread hanya ketika status benar-benar berubah. Pendekatan ini menghilangkan polling periodik yang memboroskan CPU.

// Event-driven pattern
waitqueue_wait(&obj->wq, &ctx);
// Thread sleeps until event occurs
// No polling, no busy-waiting

Runtime Userspace

KlatOS memiliki runtime userspace yang lengkap. Kernel mampu menjalankan executable ELF 64-bit melalui loader internal.

  • ELF Loader
  • Dynamic Linker
  • C Runtime
  • POSIX ABI
  • System Call Interface
  • Process Startup
  • Userspace Memory Manager

Kompatibilitas POSIX

JowoKernel mengimplementasikan sebagian besar syscall dasar yang dibutuhkan sistem UNIX modern:

// Process management
fork()  execve()  wait4()  exit()  kill()

// File I/O
open()  close()  read()  write()  lseek()
dup()  dup2()  pipe()

// Memory
mmap()  munmap()  mprotect()  brk()

// Network
socket()  socketpair()  bind()  connect()
listen()  accept4()  sendmsg()  recvmsg()

// Events
poll()  select()  epoll()

Graphics Stack

KlatOS menggunakan graphics stack modern berbasis DRM (Direct Rendering Manager):

  • DRM Device — interface ke GPU melalui /dev/dri/card0
  • Dumb Buffer — framebuffer sederhana tanpa GPU acceleration
  • Framebuffer Management — pengelolaan buffer render
  • Page Flipping — ganti frame tanpa tearing
  • VBlank Synchronization — sinkronisasi dengan refresh rate monitor
  • Zero-copy Framebuffer — sharing buffer antar proses
  • Memory Mapped Graphics — akses framebuffer via mmap
  • Event-driven Rendering — render hanya saat ada perubahan

Kompatibilitas libdrm

Untuk meningkatkan kompatibilitas, KlatOS menyediakan implementasi clean-room dari API libdrm:

// Available API
drmIoctl()
drmModeGetResources()
drmModeCreateDumbBuffer()
drmModeMapDumbBuffer()
drmModeAddFB()
drmModePageFlip()

Aplikasi dapat menggunakan API standar tanpa perlu mengetahui implementasi internal kernel.

Desktop Platform

Desktop KlatOS dibangun sebagai kumpulan proses independen yang saling berkomunikasi melalui POSIX IPC:

  • Display Server — mengelola output display
  • Window Manager — mengelola jendela dan tata letak
  • Input Server — menangani keyboard, mouse, touchscreen
  • Desktop Shell — wallpaper, panel, taskbar
  • Launcher — peluncur aplikasi
  • Notification Service — sistem notifikasi
  • Session Manager — manajemen sesi login

Pendekatan ini meningkatkan stabilitas karena kegagalan satu komponen tidak secara langsung menghentikan keseluruhan desktop.
